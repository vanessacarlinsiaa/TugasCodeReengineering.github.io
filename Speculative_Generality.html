Speculative Generality

Speculative Generality terjadi ketika sebuah code memiliki fitur atau function yang tidak digunakan saat ini, tetapi disertakan karena diperkirakan mungkin akan berguna di masa depan. Hal ini dapat menciptakan kompleksitas yang tidak perlu dan mempersulit pemahaman terhadap code.

Alasan utama terjadinya "Speculative Generality":


Speculative generality sering kali muncul karena pengembang mencoba untuk memprediksi kebutuhan masa depan tanpa basis yang kuat atau permintaan yang jelas. Hal ini dapat menyebabkan overhead dalam pengembangan, pemeliharaan, dan pengujian code.

Cara Mengatasi

Untuk mengatasi speculative generality, penting untuk fokus pada kebutuhan yang sebenarnya dan menghindari menambahkan fitur atau function yang tidak dibutuhkan saat ini. Diperlukan pendekatan yang lebih tangkas dan iteratif dalam pengembangan software.

>
Sebelum Refactor
                            
public class SpeculativeGeneralityExample {
    public void doSomething() {
        // implementasi yang kompleks dan tidak dibutuhkan saat ini
        // karena diperkirakan mungkin akan berguna di masa depan
    }
}
Setelah Refactor
                            
// Hanya menyertakan fitur yang dibutuhkan saat ini
public class RefactoredSpeculativeGeneralityExample {
    public void doSomethingNeededNow() {
        // implementasi yang diperlukan saat ini
    }
}

Dalam penyelesaian code yang disajikan di atas, kita membuang fitur yang tidak dibutuhkan saat ini dan hanya menyertakan apa yang benar-benar diperlukan. Hal ini membuat code menjadi lebih sederhana dan mudah dipahami.
