Data Class

Data class adalah class yang hanya berfungsi sebagai wadah untuk menyimpan data tanpa memiliki logika bisnis yang signifikan. Dispensable bad code smell terjadi ketika ada data class yang sebenarnya tidak diperlukan dan hanya menambah kompleksitas code tanpa memberikan nilai tambah yang nyata.

Alasan utama terjadinya "Data Class":

Data class yang tidak diperlukan dapat memperumit struktur code dan membuatnya sulit dipahami. Hal ini dapat menyebabkan overhead dalam pengembangan, pemeliharaan, dan debugging code.

Cara Mengatasi

  • Evaluasi apakah data class tersebut benar-benar diperlukan. Jika tidak, pertimbangkan untuk menghapusnya.

  • Jika data class memang diperlukan, pastikan bahwa ia memiliki tanggung jawab yang jelas dan memanggil hanya pada kasus penggunaan yang relevan.
Sebelum Refactor
                            
public class Person {
    private String name;
    private int age;
    private String address;

    public Person(String name, int age, String address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }

    // getters and setters
}

Sebelum perubahan, kita memiliki class Java bernama Person yang memiliki tiga properti: name, age, dan address. Class ini memiliki konstruktor untuk menginisialisasi nilai properti tersebut, serta getter dan setter untuk akses ke properti-properti tersebut. Namun, tidak ada logika tambahan di dalam class tersebut selain pengaturan dan pengambilan nilai properti.

Setelah Refactor
                            
// code ini hanya menggunakan field-field dari Person, tanpa logika tambahan.
// Jadi, mengubah Person menjadi data class lebih baik.
public record Person(String name, int age, String address) {}

Setelah melakukan perubahan, kita mengubah class Person menjadi sebuah data class menggunakan fitur baru dari Java 16 yaitu record. Sekarang, Person direpresentasikan oleh satu baris code dengan menggunakan record. Record secara otomatis menyediakan konstruktor, getter, dan method lainnya untuk properti-properti yang diberikan pada konstruktor. Ini mengurangi boilerplate code dan membuat code lebih ringkas dan mudah dibaca. Dengan perubahan ini, code menjadi lebih sederhana dan mudah dipahami. Kita masih bisa mengakses properti name, age, dan address dengan cara yang sama seperti sebelumnya, namun dengan code yang lebih efisien.
