Comments

Comments adalah kondisi di mana code memiliki banyak comment yang seharusnya tidak perlu. Ini mungkin terjadi karena code yang kompleks atau sulit dipahami, sehingga pengembang merasa perlu untuk menambahkan banyak comment untuk menjelaskan apa yang dilakukan oleh code tersebut.

Alasan utama terjadinya "Comment":


Biasanya, keberadaan comment yang berlebihan menandakan bahwa code itu sendiri tidak cukup bersih atau jelas. comment yang terlalu banyak dapat membingungkan pengembang lain yang mencoba memahami code, atau bahkan dapat menjadi usaha untuk menyembunyikan masalah dalam desain atau struktur code.

Cara Mengatasi

  • Usahakan untuk menulis code yang bersih dan mudah dipahami tanpa perlu mengandalkan comment berlebihan.
  • Gunakan nama variabel, function, dan class yang deskriptif sehingga tidak perlu menambahkan comment untuk menjelaskan tugas mereka.
  • Evaluasi apakah comment yang ada benar-benar diperlukan atau apakah mereka hanya memberikan informasi yang sudah jelas dari code itu sendiri.
  • Jika ada bagian code yang memang memerlukan penjelasan tambahan, pastikan comment yang ditambahkan singkat, jelas, dan terfokus pada penjelasan yang benar-benar diperlukan.

Mari kita lihat contoh sederhana di mana kita memiliki class Product yang bertanggung jawab untuk menyimpan informasi produk, dan class Inventory yang mengelola inventaris produk tersebut. Awalnya, class Product hanya memiliki satu atribut yaitu name. Namun, kemudian kita ingin menambahkan atribut price ke dalam class Product.

Sebelum Refactor
                            
// Inisialisasi objek 'data' dengan nilai awal 0
int data = 0;

// Perulangan untuk menghitung total nilai dari array 'values'
for (int i = 0; i < values.length; i++) {
    // Menambahkan nilai array ke dalam variabel 'data'
    data += values[i];
}

// Memeriksa apakah total nilai 'data' lebih besar dari 100
if (data > 100) {
    // Jika iya, cetak pesan bahwa nilai lebih besar dari 100
    System.out.println("Total nilai lebih besar dari 100");
} else {
    // Jika tidak, cetak pesan bahwa nilai tidak lebih besar dari 100
    System.out.println("Total nilai tidak lebih besar dari 100");
}

Sekarang, kita ingin menambahkan atribut "price" ke dalam class "Product". Namun, karena class "Inventory" menggunakan class "Product" dan hanya memiliki akses ke "name", kita harus mengubah class "Inventory" juga.

Setelah Refactor
                            
int totalNilai = 0; // Inisialisasi total nilai dengan nilai awal 0

// Menghitung total nilai dari array 'values'
for (int i = 0; i < values.length; i++) {
    totalNilai += values[i];
}

// Memeriksa apakah total nilai lebih besar dari 100
if (totalNilai > 100) {
    System.out.println("Total nilai lebih besar dari 100");
} else {
    System.out.println("Total nilai tidak lebih besar dari 100");
}

Dalam contoh di atas, comment-comment yang sebelumnya dianggap berlebihan telah dihilangkan dengan cara menambahkan nama variabel yang deskriptif dan mengurangi penjelasan yang tidak perlu. code tersebut sekarang menjadi lebih bersih dan mudah dipahami tanpa perlu mengandalkan comment yang berlebihan.
